.NET Long Answers – Full Notes

✅ 1. Multiple Inheritance (Using Interfaces) – Long Answer

Multiple inheritance means a class can inherit features from more than one parent. C# does not allow multiple inheritance using classes because it creates ambiguity (known as the diamond problem). However, C# supports multiple inheritance using interfaces, where a class can implement multiple interfaces.

An interface contains only method declarations. A class implementing the interface must define all methods.

interface A {
    void add();
}

interface B {
    void sub();
}

class C : A, B {
    public void add() {
        int a = 10, b = 20, c = a + b;
        Console.WriteLine("addition " + c);
    }

    public void sub() {
        int a = 10, b = 20, c = a - b;
        Console.WriteLine("sub " + c);
    }
}
  • Allows sharing behavior from multiple parent types.
  • Avoids ambiguity.
  • Supports abstraction and flexibility.
✅ 2. Base Class Library (BCL) – Long Answer

The Base Class Library (BCL) is a core part of the .NET Framework. It is a collection of pre-built, reusable classes that simplify software development.

BCL includes classes for:

  • Input/output operations (System.IO)
  • Data types (System)
  • Collections (System.Collections)
  • Networking (System.Net)
  • File handling
  • Database operations
  • Math & threading

Importance:

  • Reduces development time.
  • Ensures consistency across applications.
  • Works with all .NET languages.

BCL is the foundation layer of .NET applications.

✅ 3. Common Language Specification (CLS) – Long Answer

The CLS is a set of rules that all .NET languages must follow to ensure cross-language compatibility. Since .NET supports many languages (C#, VB.NET, F#, etc.), CLS makes them work together smoothly.

CLS defines rules for:

  • Naming conventions
  • Data types
  • Method signatures
  • Visibility rules

Importance:

  • Promotes language interoperability.
  • Ensures cross-language compatibility.
✅ 4. Common Type System (CTS) – Long Answer

The CTS defines all data types supported by .NET, how they behave, and how they are used and managed.

Two categories:

  • Value Types: int, float, bool
  • Reference Types: class, object, string

Importance of CTS:

  • Ensures consistency across languages.
  • Removes type conflicts.
  • Supports cross-language inheritance.
✅ 5. CLR (Common Language Runtime) – Long Answer

The CLR is the execution engine of the .NET Framework.

Core responsibilities:

  • Memory Management
  • Garbage Collection
  • Security Checking
  • JIT Compilation
  • Exception Handling
  • Thread Management

Execution Process:

  1. Code → IL
  2. IL → Machine code via JIT
  3. CLR executes machine code
✅ 6. JIT (Just-In-Time Compiler) – Long Answer

The JIT compiler converts IL code into machine code at runtime.

Types of JIT:

  • Pre-JIT
  • Econo-JIT
  • Normal JIT

Advantages:

  • Faster execution
  • Optimizes runtime performance
  • Adapts to CPU architecture
✅ 7. Garbage Collection – Long Answer

Garbage Collection (GC) automatically frees unused memory.

Benefits:

  • Prevents memory leaks
  • Removes unused objects
  • Improves stability

GC works in generations (0, 1, 2) for efficiency.

✅ 8. Framework Class Library (FCL) – Long Answer

The FCL is an extended version of the BCL, containing advanced functionality.

Includes:

  • Web libraries
  • ADO.NET (Database)
  • Graphics
  • XML
  • Windows Forms
  • ASP.NET

FCL = BCL + advanced libraries.

✅ 9. OS Role in .NET – Long Answer

The .NET Framework runs on top of the operating system.

OS provides:

  • File handling
  • Memory management
  • Process control
  • Security
  • Hardware access
✅ 10. .NET Application Layer – Long Answer

This is the top layer of .NET where applications are created.

Examples:

  • Windows Forms
  • ASP.NET
  • Console Applications
✅ 11. Programming Languages in .NET – Long Answer

.NET supports languages like C#, VB.NET, and F#.

Compilation Process:

  1. Source code → IL
  2. IL → executed by CLR using JIT

Advantages:

  • Cross-language support
  • Reusable libraries
  • Multi-language flexibility
